Class deCarta.Core.Geocoder

The Geocoder object is a static object that provides an interface to execute Geocoding and Reverse Geocoding using deCarta's WebServices API.

  • #reverseGeocode - Used to translate a latitude and longitude coordinate into an approximate address.

<static> deCarta.Core.Geocoder.geocode(address, callback)
Translate an address into a list of Positions (latitude and longitude coordinates). Since user input may often be incomplete, it is important to be able to accept the multiple matches that can be returned from a geocode request. Geocoding is an asynchronous request made to the DDS Web Services, and thus requires a callBack function to catch the returned results. The application should provide a callBack function that has the following prototype: 
function geocodeCallBack(results)
Where 
results
is an array, with each element of the array having the following structure: 
{
  "Address": deCarta.Core.StructuredAddress,
  "Position": deCarta.Core.Position,
  "matchType": "string",
  "accuracy": float
}
Parameters:
{Address} address
StructuredAddress or FreeFormAddress to be geocoded

Note: This can be EITHER an instance of a deCarta.Core.StructuredAddress or deCarta.Core.FreeFormAddress class OR an equivalent structure created as an inline object.

An inline StructuredAddress can be implemented by defining an inline object with one or more of these fields:

  • buildingNumber: (optional) Address number for this location.
  • countryCode: Country, formatted as ISO 3166-1 alpha-2 (2 digit) country code. See http://developer.decarta.com/docs/read/bada_api/Features for supported countries.
  • countrySecondarySubdivision: (optional) County (or equivalent) for this location.
  • countrySubdivision: (optional) Sub-country administrative division (ie the state, province, or region) for this location.
  • countryTertiarySubdivision: (optional) Additional subdivision (when relevant)
  • landmark
  • locale: Language, formatted as ISO 639 Language code. See http://developer.decarta.com/docs/read/bada_api/Features for supported languages.
  • municipality: (optional) City, town, village, or equivalent for this location.
  • municipalitySubdivision: (optional) Recognized neighborhood, borough, or equivalent for this location.
  • postalCode: (optional) Postal code, postcode, ZIP code, or equivalent numerical code for this location.
  • speedLimit: [Not currently documented]
  • street: (optional) Name and designation of the street (ie Main St) for this location.
  • streetNameAndNumber: [Not currently documented]
  • type: Must be set to 'structured' to create an inline structured address object

An inline FreeFormAddress can be implemented by defining these fields:

  • (string) address: The free-form address
  • locale: Language, formatted as ISO 639 Language code. See http://developer.decarta.com/docs/read/bada_api/Features for supported languages.
  • type: Must be set to 'freeForm' to create an inline Free Form address object

{function} callback
The callback function that will receive the results of the geocode operation

{Exception}
Invalid or missing address object passed as parameter
{Exception}
Invalid or missing callback function
<static> deCarta.Core.Geocoder.reverseGeocode(position, callback, preference)
Translate a Position (latitude, longitude coordinate) into an Address. Reverse geocodes are always approximated to the best possible address within the range of addresses available in the map data. Reverse geocoding is an asynchronous request made to the DDS Web Services, and thus requires a callBack function to catch the returned results. The application should provide a callBack function that has the following prototype: 
 function reverseGeocodeCallBack(result)
If a valid reverse geocode can be made, the resulting position will be passed to the callBack in an result object. The structure of the result object is as follows: { "Address": deCarta.Core.StructuredAddress, "Position": deCarta.Core.Position } A failed reverse geocode will return an empty Address (result.Address.toString() == "").
Parameters:
{deCarta.Core.Position} position
The position to reverse geocode. This can be passed as an instance of a deCarta.Core.Position class, or as an inline-object with the following fields:
  • {float} lat: Latitude of the position
  • {float} lat: Longitude of the position
{function} callback
Function that will handle the result of the reverse geocode.
preference
Throws:
{Exception}
Invalid or missing callback function
